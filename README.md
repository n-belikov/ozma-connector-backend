## Коннекторы Озмы

### Необходимые требования
- php:~7.4
- mariadb\mysql
- crontab

### Общие настройки
Вся информация по настройкам лежит в `.env.example`

### Общее по коду
Для синхронизации предусмотрены две команды:
- `ozma:sync` - выгружает все данные с коннекторов и выгружает их в саму озму
- `ozma:sync-stages` - получает состояние сущностей в самой озме и работает с ними. На данный момент только вызывается 
событие:
```php
$event = new OrderStageChangeEvent(
    \App\Models\Ozma\Order $order, // Заказ, который лежит в промежуточной БД 
    \App\Domain\Ozma\OzmaOrderBoard $orderBoard // Сущность из озмы
);
```

В проекте присутствуют мапперы данных. Зачем? Они помогают переводить данные из одного вида в другой.\
Их настройка происходит в `\App\Providers\MapperServiceProvider`\
Добавление коннекторов и их настройка происходит в `\App\Providers\ConnectorServiceProvider`\
Для общения с базой данных были написаны простые репозитории. Они необходимы для возможной п одмены реализации общения 
с базой данных.

#### Enum
Проект использует версию php 7.4 и в нём ещё не были введены enum'ы. Решением данной проблемы был вариант создания enum'ов
через классы. Пример: ConnectorType. Таким образом мы явно указываем те данные которые должны приходить, а не только строку.
